Как получить Google OAuth Client ID

  1. Откройте Google Cloud Console
  2. Создайте новый проект или выберите существующий
  3. Перейдите в APIs & Services → Library
  4. Найдите и включите Google Calendar API
  5. Перейдите в APIs & Services → OAuth consent screen, настройте экран (можно выбрать External)
  6. Перейдите в APIs & Services → Credentials
  7. Нажмите Create Credentials → OAuth 2.0 Client ID
  8. Выберите тип Web application
  9. В разделе Authorized JavaScript origins добавьте адрес, с которого открыто приложение, например:
    http://localhost:8080 (для локального запуска)
    https://your-domain.com (при публичном размещении)
  10. Нажмите Create и скопируйте Client ID
💡 Для локального запуска: Сохраните этот HTML-файл и запустите через терминал:
python3 -m http.server 8080
Затем откройте http://localhost:8080/calendar-overlap-analyzer.html
⚠️ Инструмент использует Google Calendar FreeBusy API — он видит, заняты ли люди в конкретное время, но не видит названия встреч. Данные доступны только для людей из вашей Google Workspace организации или тех, кто поделился статусом занятости с вами.
